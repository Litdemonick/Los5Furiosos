Buenas Prácticas de Desarrollo

1. Uso de Git y GitHub

  • Crear ramas por funcionalidad o tarea (feature/..., fix/...).
  • Hacer commits frecuentes, claros y descriptivos.
  • Usar pull requests para integrar cambios a la rama principal.
  • Revisar código de compañeros antes de hacer el merge.
  • Mantener la rama main o develop estable y funcional.

2. Uso de GitKraken

  • Visualizar ramas y commits de forma gráfica.
  • Hacer merge o rebase de forma clara.
  • Resolver conflictos con el editor visual de GitKraken.
  • Sincronizar cambios frecuentemente con el repositorio remoto.

3. Documentación del Proyecto

  • Usar un archivo README.md con descripción, instalación y uso.
  • Organizar la documentación en carpetas (/docs).
  • Incluir diagramas, ejemplos de código y guías rápidas.
  • Explicar dependencias y configuración inicial.

4. Buenas Prácticas Generales

  • Seguir convenciones de estilo en el código (nombres claros, consistencia).
  • Escribir comentarios útiles y concisos.
  • Usar control de versiones para todo el proyecto (no trabajar fuera de Git).
  • Actualizar dependencias y librerías de manera controlada.
  • Realizar pruebas antes de subir cambios a la rama principal.

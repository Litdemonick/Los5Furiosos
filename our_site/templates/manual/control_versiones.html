Control de Versiones — Guía práctica

Resumen de comandos y flujo de trabajo Git + pasos básicos en GitKraken

Introducción Instalar & configurar Flujo (feature → develop → main) Comandos útiles GitKraken Buenas prácticas

1. Introducción

Git es un sistema de control de versiones distribuido. Permite trabajar en paralelo, mantener el historial y colaborar mediante ramas y merges. En este documento veremos un flujo común: feature branches → develop → main.

2. Instalar y configurar Git

Comandos mínimos para empezar:

git --version
# configurar identidad (solo la primera vez)
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
# opcional: elegir editor y colores
git config --global core.editor "code --wait"
git config --global color.ui auto

3. Flujo de trabajo recomendado

Ejemplo de flujo para cada tarea o feature:

  1. Actualizar la rama develop desde remoto.
  2. Crear una rama de trabajo basada en develop (ej. feature/mi-feature).
  3. Desarrollar, commitear frecuentemente en local.
  4. Hacer push de la rama y abrir Pull Request a develop.
  5. Revisiones, tests y merge a develop. Luego integrar a main mediante release o hotfix cuando corresponda.

Comandos paso a paso

# situarse en develop y actualizar
git checkout develop
git pull origin develop

# crear rama feature y moverse a ella
git checkout -b feature/mi-feature

# trabajar, agregar y commitear
git add archivo1 archivo2
git commit -m "feat: agregar X"

# subir la rama al remoto
git push -u origin feature/mi-feature

4. Comandos útiles y atajos

  • git status — ver cambios
  • git diff — ver diferencias
  • git log --oneline --graph --all — historial compacto
  • git rebase develop — rebasear tu rama sobre develop (mantiene historial lineal)
  • git merge --no-ff feature/mi-feature — merge con commit de merge
  • git cherry-pick <commit> — traer commit específico a otra rama
  • git stash — guardar cambios temporales (ver git stash pop)

Resolver conflictos básicos

Si hay conflictos al hacer merge o rebase:

# después de conflicto editar archivos en conflicto
git add archivo-resuelto
# si estabas rebaseando
git rebase --continue
# si estabas mergeando
git commit -m "fix: resolver conflicto"

5. GitKraken — pasos rápidos

GitKraken es una interfaz gráfica que simplifica ramas, merges y rebases. Pasos para sincronizar tu rama con develop:

  1. Abrir el repo en GitKraken.
  2. En la lista de ramas seleccionar develop y hacer Fetch.
  3. Checkout en tu rama (doble click).
  4. Arrastrar tu rama hacia develop para crear un Merge o usar el botón Merge en la barra lateral.
  5. Resolver conflictos desde el editor integrado si aparecen.
  6. Hacer Push desde GitKraken para subir cambios al remoto.

Consejo: usa la vista de Graph para entender la historia visualmente y evita hacer merges desde la línea de tiempo si no estás seguro — mejor hazlo desde el menú de Merge o Pull Request.

6. Buenas prácticas

  • Commits pequeños y con mensajes claros: type(scope): mensaje (ej. feat(api): agregar endpoint de login).
  • Actualizar tu rama con develop frecuentemente para reducir conflictos.
  • Revisiones de código (PR) con checklist: tests, lint, impacto en API, documentación.
  • No subir secretos: usa variables de entorno o archivos ignorados con .gitignore.
  • Agregar un archivo README.md y guías de contribución (CONTRIBUTING.md).

Atajos rápidos

# clonar repo
git clone git@github.com:org/repo.git

# crear y pushear rama
git checkout -b fix/bug-123
git push -u origin fix/bug-123

# actualizar develop y rebasear
git checkout develop
git pull origin develop
git checkout feature/mi-feature
git rebase develop

# eliminar rama remota
git push origin --delete feature/mi-feature
